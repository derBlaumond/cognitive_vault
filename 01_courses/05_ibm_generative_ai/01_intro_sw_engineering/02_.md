# 📘 Intro to Software Development — 정리 노트

## 1️⃣ 소프트웨어 개발 개요 (Introduction to Software Development)

- **소프트웨어 개발(Software Development)**은 **요구 분석 → 설계 → 코딩 → 테스트 → 배포 → 유지보수**로 이어지는 일련의 과정이다.
    
- 목표는 사용자의 요구를 만족시키는 **신뢰성 있고 유지보수 가능한 프로그램**을 만드는 것.
    
- 모든 개발 프로세스는 **SDLC(Software Development Life Cycle)** 기반으로 구조화됨.
    

### 주요 목적

1. **요구사항 충족:** 고객이 진정으로 원하는 기능을 구현.
    
2. **품질 유지:** 오류를 줄이고 확장성을 확보.
    
3. **비용/시간 효율:** 단계별 계획으로 리스크 최소화.
    

### 소프트웨어 개발의 기본 단계

|단계|설명|
|---|---|
|**Requirements (요구사항 수집)**|고객의 목표, 제약조건, 환경 파악|
|**Design (설계)**|시스템 구조, 데이터 흐름, UI/UX 설계|
|**Implementation (구현)**|실제 코딩 수행|
|**Testing (테스트)**|오류 탐색, 기능 검증|
|**Deployment (배포)**|실제 사용자 환경에 적용|
|**Maintenance (유지보수)**|오류 수정 및 기능 개선|

## 2️⃣ 프로그래밍 언어의 분류

### 🔹 컴파일 언어(Compiled Language)

- **정의:** 전체 소스코드를 **컴파일러(Compiler)** 가 기계어로 번역 → 실행 파일 생성.
    
- **특징:**
    
    - 실행 속도가 빠름.
        
    - 실행 전 전체 코드 검사 → 오류 사전 감지.
        
    - 예시: **C, C++, Java, Go**
        
- **장점:** 최적화된 실행, 빠른 성능.
    
- **단점:** 수정 후 재컴파일 필요, 플랫폼 의존적.
    

### 🔹 인터프리터 언어(Interpreted Language)

- **정의:** 코드가 **한 줄씩 해석되어 실행**됨.
    
- **특징:**
    
    - 실행 파일 생성 X.
        
    - 코드 수정 즉시 테스트 가능.
        
    - 예시: **Python, JavaScript, PHP**
        
- **장점:** 개발 속도 빠름, 디버깅 용이.
    
- **단점:** 실행 속도 느림, 배포 시 코드 노출 가능.
    

### 비교 요약

|구분|컴파일 언어|인터프리터 언어|
|---|---|---|
|실행 방식|전체 번역 후 실행|한 줄씩 해석 실행|
|속도|빠름|느림|
|수정 용이성|낮음|높음|
|예시|C, Java|Python, JS|

## 3️⃣ 프로그래밍 언어 선택 기준

- **프로젝트 목적**과 **운영 환경**이 언어 선택의 핵심이다.
    
- 예시:
    
    - 웹 클라이언트 중심 → JavaScript
        
    - 백엔드 서버 중심 → Java, Python, Node.js
        
    - 모바일 앱 → Swift, Kotlin
        
    - 시스템 개발 → C/C++

## 4️⃣ 고급언어 vs 저급언어 (High-level vs Low-level)

### 고급언어 (High-level Language)

- 사람이 읽기 쉬운 구문 사용.
    
- 플랫폼 독립적.
    
- 예시: Python, Java, SQL
    

### 저급언어 (Low-level Language)

- 하드웨어 명령어와 직접 대응.
    
- 효율은 높지만 사람이 읽기 어려움.
    
- 예시: Assembly, Machine Code
    

---

## 5️⃣ SQL과 쿼리 언어 (Query Languages)

- **쿼리(Query)**: 데이터베이스에 정보를 요청하는 명령어.
    
- 대표 언어: **SQL (Structured Query Language)**
    
- **CRUD** 연산: Create, Read, Update, Delete  
    → 데이터 조작의 핵심 네 가지 기능.
    

|명령어|기능|
|---|---|
|SELECT|데이터 조회|
|INSERT|새 데이터 추가|
|UPDATE|기존 데이터 수정|
|DELETE|데이터 삭제|

- SQL 외에도 **AQL, CQL, Datalog, DMX** 등 다양한 쿼리 언어가 존재.
    
- SQL은 관계형 DB, NoSQL은 비정형 데이터 처리에 사용.

## 6️⃣ 어셈블리 언어 (Assembly Language)

- **저급언어(Low-level)** 의 대표 예시.
    
- 기계어(0과 1)를 사람이 이해할 수 있는 **기호(mnemonics)** 로 표현.
    
- 예시: `LDA`, `STA`, `ADD`, `INP`, `OUT`
    
- **특징:**
    
    - 하드웨어 구조에 따라 언어가 다름.
        
    - **Assembler** 를 통해 기계어로 변환.
        
    - 1개의 명령문 ↔ 1개의 기계 명령.
        
- 장점: 속도와 효율 높음.
    
- 단점: 복잡하고 유지보수 어려움.

## 7️⃣ 코드 조직화 방법 (Code Organization)

### 주요 방법:

1. **Flowchart (순서도)**
    
    - 알고리즘의 단계별 흐름을 시각화.
        
    - 사용 도형:
        
        - **Start/End:** 타원형
            
        - **Process:** 사각형
            
        - **Decision:** 마름모
            
        - **Input/Output:** 평행사변형
            
        - **Connector:** 화살표
            
    - 장점: 직관적이고 초보자에게 유용.
        
2. **Pseudocode (의사코드)**
    
    - 실제 코드가 아닌 **논리적 설명문**으로 프로그램 절차를 표현.
        
    - 언어 독립적이며 협업 시 이해도 높음.
        
    - 예시:
        
        ```
        If number % 2 == 0
            Print "Even"
        Else
            Print "Odd"
        ```
        

|구분|Flowchart|Pseudocode|
|---|---|---|
|표현방식|시각적 (도형)|텍스트 기반|
|장점|흐름 이해 용이|언어 독립적|
|권장 용도|소규모 로직|대규모 알고리즘|

## 8️⃣ 프로그래밍 논리 (Branching & Looping)

### Boolean 논리

- **True / False** 값을 기반으로 조건 판단.
    
- 변수는 상황에 따라 값이 변함.
    

### Branching (분기)

- 조건에 따라 코드의 실행 경로를 결정.
    
- 대표 구문:
    
    - `if`, `if-else`, `switch`, `goto`
        
- 예:
    
    ```python
    if score > 90:
        grade = 'A'
    else:
        grade = 'B'
    ```
    

### Looping (반복)

- 조건이 만족될 때까지 코드 반복 실행.
    
- 종류:
    
    - `while`
        
    - `for`
        
    - `do-while`
        
- 예:
    
    ```python
    for i in range(5):
        print(i)
    ```

## 9️⃣ 식별자와 컨테이너 (Identifiers & Containers)

### 식별자(Identifier)

- 프로그램 내 변수, 함수, 클래스 등을 구분하는 이름.
    
- 두 종류:
    
    - **Constant (상수):** 값이 변하지 않음.
        
    - **Variable (변수):** 실행 중 값 변경 가능.
        

### 컨테이너(Container)

- 여러 데이터를 한 번에 관리하기 위한 구조.
    
- 예: **List, Dictionary, Set, Array**
    
- 장점:
    
    - 데이터 접근/정렬 용이.
        
    - 반복문과 결합되어 효율적인 데이터 처리 가능.

## 🔟 내부자 관점 (Insider’s Viewpoint)

- **전문가 의견:**
    
    - 언어 선택은 프로젝트 규모와 성능 요구에 따라 달라짐.
        
    - **컴파일 언어**는 성능 중심,  
        **인터프리터 언어**는 유연성과 생산성 중심.
        
    - **객체지향(Object-Oriented)** 은 실세계 모델링에 적합하나,  
        과도한 추상화는 오히려 복잡성을 증가시킬 수 있음.
        
    - **절차적 언어(Procedural)** 는 수학적이고 구조적 사고에 적합.
