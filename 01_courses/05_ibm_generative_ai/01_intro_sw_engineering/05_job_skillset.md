## 1. What Does a Software Engineer Do?

소프트웨어 엔지니어는 **공학, 수학, 컴퓨팅 지식을 활용해 현실의 문제를 해결하는 소프트웨어를 설계하고 개발하는 전문가**이다.  
분석적 사고를 좋아하고 문제 해결을 즐기는 사람이라면, 소프트웨어 엔지니어링은 적합한 진로가 될 수 있다.

### 근무 형태 및 역할

소프트웨어 엔지니어는 다음과 같은 환경에서 일할 수 있다:

- **Off-the-shelf 소프트웨어**: 범용 제품 소프트웨어
- **Bespoke 소프트웨어**: 특정 고객의 요구에 맞춘 맞춤형 프로그램
- **Internal 소프트웨어**: 내부 직원용 시스템

### 주요 업무 예시

- 사용자 요구사항 분석 및 설계
- 코드 작성 및 기능 테스트
- 새 소프트웨어 프로그램 평가 및 성능 최적화
- 기존 시스템 유지보수 및 업데이트
- 코드 문서화 및 팀원 이해를 위한 주석 작성
- 고객 또는 사용자에게 시스템 시연
- DevOps 엔지니어의 경우, 코드 통합 및 배포도 수행

## 2. A Day in the Life of a Software Engineer
### 일상 루틴

- **Daily Standup 미팅**: 전날 성과와 오늘의 작업 목표를 공유한다.
- 멘토 Jonathan의 피드백을 받아 코드 최적화를 진행한다.
- **Focus Time**에는 코드 성능 개선에 집중하고,
    - 성능 비교 분석 소프트웨어를 이용해 개선 효과를 검증한다.
    - 수정 후 새로운 merge request를 제출하고 리뷰 요청을 한다.

### 협업 및 학습

- 마케팅 부서에서 새로운 기능 요청 → React 기반 대시보드 개발 과제 부여.
- 처음 사용하는 기술(React)이라도 실무에서 배우며 개발에 기여한다.
- 팀 내 시니어 엔지니어 Marie가 기존 React 프로젝트를 공유해 참고할 수 있도록 돕는다.
- 오후에는 버그 리포트를 받고 문제를 재현 → 수정 → 테스트 케이스 작성 → 통과 후 병합 요청 제출.

## 3. Skills Required for Software Engineering

소프트웨어 엔지니어에게 필요한 기술은 **Hard Skill(기술적 역량)** 과 **Soft Skill(인성적 역량)** 로 나뉜다.

### Hard Skills (하드 스킬)

하드 스킬은 **측정 가능하고 학습을 통해 습득하는 실무 기술**이다.  
#### 주요 하드 스킬

- **프로그래밍 언어** (Java, Python, C#, Ruby 등)
- **버전 관리 시스템** (Git 등)
- **클라우드 컴퓨팅 및 배포 기술**
- **테스트와 디버깅 능력**
- **CI/CD (지속적 통합/배포)**
- **데이터베이스 아키텍처 이해**
- **문제 해결 및 소프트웨어 설계 능력

#### 개발 단계별 기술

1. 분석 및 설계: 사용자 요구사항을 파악해 설계안 도출
2. 개발: 언어와 프레임워크를 사용해 구현
3. 테스트: 기능 및 사용성 검증
4. 디버깅: 오류 원인 탐색 및 수정
5. 배포: 셸 스크립트, 컨테이너(Docker), CI/CD 도구 사용
6. 모니터링 및 문제 해결

### Soft Skills (소프트 스킬)

소프트 스킬은 **태도와 성격에 기반한 비기술적 역량**이다. 정량화하기 어렵지만, 역할 전환과 산업 간 이동이 용이한 강점이 있다.

#### 주요 소프트 스킬

- **팀워크**: 스쿼드 단위 협업, 페어 프로그래밍 등
- **커뮤니케이션 능력**: 동료, 관리자, 클라이언트, 사용자와의 소통
- **시간 관리**: 마감 준수, 타임존 차이 고려한 협업
- **문제 해결 능력**: 설계, 개발, 테스트 전 과정에서 요구됨
- **적응력**: 요구사항 변경, 팀 이동, 기능 수정 등에 유연하게 대응
- **피드백 수용력**: 코드 리뷰, 멘토 피드백, 사용자 피드백을 열린 태도로 받아들이기

## 4. Career Paths in Software Engineering

소프트웨어 엔지니어의 커리어는 **기술 중심(Technical)** 또는 **관리 중심(Managerial)** 으로 발전한다.

### 1) 기술 경로

- **Junior Software Engineer / Associate Developer**
    
    - 작은 기능 구현, 코드 작성 및 테스트 중심
    - 멘토의 지도로 성장 단계
        
- **Software Engineer**
    
    - 독립적 업무 수행, 서브 프로젝트 단위 책임
    - 언어 및 SDLC(Software Development Lifecycle) 이해 요구
        
- **Senior Software Engineer**
    
    - 전체 프로젝트와 코드베이스에 관여
    - 멘토링, 코드 리뷰 담당
    - 복잡한 문제 해결 능력 요구
        
- **Staff / Principal Engineer / Technical Architect**
    
    - 시스템 아키텍처 설계 및 기술 방향성 결정
    - 조직 내 기술 표준 수립 및 효율성 관리

### 2) Manager 급 경로

- **Technical Lead**
    - 개발팀 관리, 개발 프로세스 감독
    - 전체 개발 생명주기 책임
        
- **Engineering Manager → Director of Engineering
    - 전략적 우선순위 설정, 인력계획, 장기 목표 수립
        
- **CTO (Chief Technology Officer)**
    - 기술 비전 및 R&D 총괄, 회사의 기술 방향성 책임

### 전환 가능한 역할

- 고객과의 소통을 좋아하면 **Technical Sales / Support**
- 데이터 분석을 선호하면 **Data Engineer / Data Scientist
- 디버깅에 흥미가 있다면 **Software Tester

## 5. Code of Ethics

IEEE-CS와 ACM이 공동 개발한 **소프트웨어 공학 윤리강령(Code of Ethics)** 은 소프트웨어 엔지니어가 사회적 책임감을 가지고 일하기 위한 8가지 원칙을 제시한다. 이 코드는 법적 규정보다는 **개인적 양심과 전문적 판단을 보완하는 지침**으로 작용한다.

### 윤리강령 8원칙

1. **==Public== (공익)**
    - 안전성, 공정성, 접근성, 정직성을 지키며 공공의 이익을 우선한다.
        
2. **==Client/Employer== (고객·고용주)**
    - 이해충돌을 피하고, 정직하게 업무를 수행하며, 기밀 유지.
        
3. **==Product== (제품)
    - 품질, 비용, 일정의 균형을 고려해 신뢰할 수 있는 제품 개발.
        
4. **==Judgement== (판단)**
    - 독립성과 객관성을 유지하고 부정행위를 하지 않는다.
        
5. **==Management== (관리)
    - 리더는 위험 최소화, 보안 절차 강화, 합리적 보상 보장.
        
6. **==Profession== (직업정신)
    - 전문성 유지, 동료 비방 금지, 직업의 명예를 보호.
        
7. **==Colleagues== (동료)
    - 존중과 공정, 타인의 성과를 가로채지 않으며 동료의 윤리 준수를 격려.
        
8. **==Self== (자기개발)
    - 지속적인 학습과 품질 중심의 직업윤리 유지.
